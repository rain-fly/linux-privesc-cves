# 漏洞汇总与排查建议

## 1. Linux Kernel Copy Fail 本地提权漏洞

- **漏洞名称**：Copy Fail
- **CVE 编号**：CVE-2026-31431
- **漏洞类型**：本地权限提升
- **风险等级**：高危
- **披露来源**：`Linux.md`、`Linux内核CVE-2026-31431Copy_Fail本地提权漏洞风险预警.docx`、`关于Linux 内核 Copy Fail 存在本地提权漏洞的预警通报.pdf`

### 漏洞概述
该漏洞是 Linux 内核中的一个本地提权漏洞，由 Xint Code 研究团队披露。攻击者可利用 Linux 内核加密子系统中的逻辑缺陷，在无需竞争条件的情况下稳定完成提权。

### 漏洞成因
漏洞源于 Linux 内核加密子系统中 `authencesn` 模块的逻辑缺陷。攻击者结合 `AF_ALG` 套接字与 `splice()` 系统调用后，可将目标文件的页缓存引用暴露到可写散列表中，进而对任意可读文件页缓存实施精确的 4 字节写入。

### 利用条件
- 攻击者具备本地低权限账号
- 目标系统运行受影响内核版本
- 系统启用或允许加载 `algif_aead` 模块
- 容器场景下，容器具备可触发相关系统调用的能力

### 主要危害
- 本地普通用户提权至 `root`
- 篡改 `setuid` 程序页缓存实现权限提升
- 容器逃逸
- 跨租户攻击
- 影响 CI/CD Runner、多用户主机、Serverless 等隔离场景

### 受影响范围
- 漏洞引入提交：`72548b093ee3`
- 修复提交：`a664bf3d603d`
- 受影响时间跨度：自 2017 年起
- 默认配置下启用或可按需加载 `algif_aead` 的系统均存在风险

### 文档中明确提到的受影响版本
- Ubuntu 24.04 LTS 及以下
- Amazon Linux 2023 及以下
- RHEL 8 / 9 / 10 及以下
- SUSE 16 及以下
- Debian / Arch / Fedora / Rocky / Alma / Oracle / OpenEuler 等同期内核版本
- 长期支持中的受影响内核分支：`6.12`、`6.6`、`6.1`、`5.15`、`5.10`

### 不受影响或已修复版本
- 主线内核 `>= 7.0`
- 稳定版 `>= 6.18.22`
- 稳定版 `>= 6.19.12`

### 排查方法
```bash
uname -r
lsmod | grep -w algif_aead
lsof | grep AF_ALG
ls /etc/modprobe.d/*algif* 2>/dev/null
```

### 修复与缓解建议
1. 优先升级内核到安全版本
2. 如暂时无法升级，禁用 `algif_aead` 模块
3. 清理页缓存，降低残留利用窗口
4. 容器环境通过 Seccomp 等策略阻断 `AF_ALG`

### 临时缓解命令
```bash
echo "install algif_aead /bin/false" > /etc/modprobe.d/disable-algif-aead.conf
rmmod algif_aead 2>/dev/null || true
sync && echo 3 > /proc/sys/vm/drop_caches
```

## 2. Linux Kernel Dirty Frag 本地提权漏洞

- **漏洞名称**：Dirty Frag
- **CVE 编号**：文档中未给出统一 CVE 编号
- **漏洞类型**：本地权限提升
- **风险等级**：高危
- **披露来源**：`【已复现】Linux Kernel Dirty Frag 本地权限提升漏洞.pdf`

### 漏洞概述
该漏洞是 Linux 内核中的本地提权漏洞，公开材料显示其与 `xfrm-ESP` 和 `RxRPC` 相关逻辑缺陷有关，PoC 已公开。

### 漏洞成因
公开材料指出，该问题与内核中外部页碎片、共享 frag 处理逻辑有关。修复思路是在 `esp_input` / `esp6_input` 相关处理路径中支持 `SKBFL_SHARED_FRAG` 标记，确保 `splice` 等路径不会错误复用外部固定页。

### 利用条件
- 攻击者具备本地低权限账号
- 系统运行未修复内核
- 相关模块或功能路径可达：`esp4`、`esp6`、`rxrpc`

### 主要危害
- 本地普通用户提权
- 影响多发行版 Linux 系统

### 受影响信息
文档中提到以下已知受影响环境示例：
- Ubuntu 24.04.4 (`6.17.0-23-generic`)
- RHEL 10.1 (`6.12.0-124.49.1.el10_1.x86_64`)
- openSUSE Tumbleweed (`7.0.2-1-default`)
- CentOS Stream 10 (`6.12.0-224.el10.x86_64`)
- AlmaLinux 10 (`6.12.0-124.52.3.el10_1.x86_64`)
- Fedora 44 (`6.19.14-300.fc44.x86_64`)

### 修复信息
- 主线修复提交：`f4c50a4034e62ab75f1d5cdd191dd5f9c77fdff4`
- 具体是否已修复，需要结合发行版 backport 情况判断

### 排查方法
```bash
uname -r
lsmod | egrep '^(esp4|esp6|rxrpc)\b'
grep -R 'install esp4 /bin/false\|install esp6 /bin/false\|install rxrpc /bin/false' /etc/modprobe.d/ 2>/dev/null
```

### 修复与缓解建议
1. 关注发行版内核补丁与 backport 公告
2. 暂时禁用 `esp4`、`esp6`、`rxrpc` 模块
3. 评估禁用后对 IPsec ESP、RxRPC 等能力的影响

### 临时缓解命令
```bash
cat > /etc/modprobe.d/dirtyfrag.conf <<'EOF'
install esp4 /bin/false
install esp6 /bin/false
install rxrpc /bin/false
EOF

rmmod esp4 esp6 rxrpc 2>/dev/null || true
```

## 3. cPanel 登录流程认证绕过漏洞

- **漏洞名称**：cPanel 登录流程认证绕过
- **CVE 编号**：CVE-2026-41940
- **漏洞类型**：认证绕过
- **风险等级**：严重
- **CVSS 3.1**：9.8
- **披露来源**：`【已复现】cPanel 登录流程认证绕过漏洞（CVE-2026-41940）.pdf`

### 漏洞概述
该漏洞影响 cPanel 与 WHM 登录流程，可导致未授权攻击者绕过认证。公开材料显示该漏洞细节与 PoC 已经披露，风险较高。

### 漏洞成因
目录内文档没有给出完整源码级根因分析，但明确指出问题位于登录认证流程中，属于认证绕过类漏洞。

### 利用条件
- 目标暴露 cPanel/WHM 管理登录入口
- 版本处于受影响范围
- 攻击者可远程访问相关端口

### 主要危害
- 未授权访问管理面
- 接管 cPanel / WHM 管理能力
- 进一步造成站点和服务器失陷

### 受影响版本
- `11.86.0 < 11.86.0.41`
- `11.110.0 < 11.110.0.97`
- `11.118.0 < 11.118.0.63`
- `11.126.0 < 11.126.0.54`
- `11.130.0 < 11.130.0.18`
- `11.132.0 < 11.132.0.29`
- `11.134.0 < 11.134.0.20`
- `11.136.0 < 11.136.0.5`
- `WP Squared 11.136.1 < 11.136.1.7`

### 安全版本
- `11.86.0.41`
- `11.110.0.97`
- `11.118.0.63`
- `11.126.0.54`
- `11.130.0.18`
- `11.132.0.29`
- `11.134.0.20`
- `11.136.0.5`
- `WP Squared 11.136.1.7`

### 排查方法
```bash
test -e /usr/local/cpanel/version && cat /usr/local/cpanel/version
ss -lntp | egrep ':(2082|2083|2086|2087)\b'
```

### 修复与缓解建议
1. 立即升级到官方修复版本
2. 限制 `2082/2083/2086/2087` 端口访问来源
3. 使用 VPN 或堡垒机访问管理面
4. 配置 WAF / ACL 监控并拦截异常登录流量

## 5. 结论

本目录中的漏洞资料主要覆盖两类 Linux 内核本地提权问题，以及一个 cPanel 远程认证绕过问题。其中：
- `Copy Fail` 具备极强的稳定性和隐蔽性，应优先排查
- `Dirty Frag` 需要重点结合发行版补丁状态判断
- `cPanel CVE-2026-41940` 影响面明确，若对外暴露应立即修复
