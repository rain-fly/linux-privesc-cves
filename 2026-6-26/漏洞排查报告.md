## 修复建议优先级

### 🔴 高优先级（建议立即修复）

1. **配置登录失败锁定** - 防止暴力破解攻击
2. **关闭高危端口** - 631端口（主机一）、111端口（主机二）
3. **禁用SSH root登录** - 主机二需设置 PermitRootLogin 为 no
4. **处理过期账户** - 删除或停用不再使用的账户
5. **限制su命令使用** - 主机二需配置 pam_wheel.so
6. **配置访问限制** - 主机二需限制访问IP范围
7. **禁用系统账号** - 主机二需禁用 svc_account 账号

### 🟡 中优先级（建议尽快修复）

1. **设置密码复杂度要求** - 主机一需配置 pam_pwquality.so
2. **配置终端自动退出** - 设置 TMOUT=600
3. **限制密码重用** - 配置 remember=5
4. **配置审计规则** - 收集各类安全事件
5. **配置远程日志** - 将日志发送到远程日志主机
6. **卸载不必要软件** - telnet、cups、bluez等
7. **禁用不必要服务** - avahi-daemon等
8. **设置umask为027** - 加强文件权限控制

## 修复操作命令参考

### 1. 配置登录失败锁定

```bash
# 编辑 /etc/security/faillock.conf
deny=3
unlock_time=300
```

### 2. 设置终端自动退出

```bash
# 编辑 /etc/profile
TMOUT=600
readonly TMOUT
export TMOUT

# 使配置生效
source /etc/profile
```

### 3. 禁用SSH root登录

```bash
# 编辑 /etc/ssh/sshd_config
PermitRootLogin no

# 重启SSH服务
systemctl restart sshd
```

### 4. 关闭高危端口

```bash
# 关闭CUPS服务（631端口）
systemctl stop cups
systemctl disable cups

# 关闭rpcbind服务（111端口）
systemctl stop rpcbind
systemctl disable rpcbind
```

### 5. 卸载不必要软件

```bash
yum remove telnet cups bluez xorg-x11-server-common
```

### 6. 配置审计规则

```bash
# 编辑 /etc/audit/rules.d/audit.rules
-a always,exit -F arch=b64 -S adjtimex,settimeofday,clock_settime -k time-change
-a always,exit -F arch=b32 -S adjtimex,settimeofday,clock_settime -k time-change
-w /etc/localtime -p wa -k time-change
-w /etc/group -p wa -k identity
-w /etc/passwd -p wa -k identity
-w /etc/gshadow -p wa -k identity
-w /etc/shadow -p wa -k identity
-w /etc/security/opasswd -p wa -k identity
-w /etc/sudoers -p wa -k scope
-w /etc/sudoers.d/ -p wa -k scope
-a always,exit -F arch=b64 -S unlink,unlinkat,rename,renameat -F auid>=1000 -F auid!=unset -k delete
-a always,exit -F arch=b32 -S unlink,unlinkat,rename,renameat -F auid>=1000 -F auid!=unset -k delete

# 加载规则
augenrules --load
```
